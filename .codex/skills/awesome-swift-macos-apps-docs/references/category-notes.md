# Category Notes

Use these notes when category choice is ambiguous.

## General rule

Prefer the category that already contains the closest comparable apps in this repository.

If English and Chinese sections are not perfectly symmetrical, follow each document's local structure instead of forcing identical placement.

## Common placements in this repository

- `AI`
  - AI chat clients
  - AI coding-agent dashboards
  - Claude/Codex/Gemini helper utilities
  - AI writing, summarization, or translation tools

- `Local LLM`
  - Local model runners
  - OpenAI-compatible local model servers
  - Tools centered on serving or installing local LLMs

- `Other Development`
  - Git identity and SSH key tools
  - HTTP debugging tools
  - Regex testers
  - Developer-focused utilities that do not fit a more specific subsection

- `Web Development`
  - Docker-based local web stacks
  - Kubernetes helpers
  - Local website and site-auditing tools

- `Terminal`
  - Terminal environments
  - Terminal multiplexers
  - SSH workspaces
  - Native shells or terminals focused on coding-agent workflows

- `Window Management`
  - Tiling
  - Snapping
  - Rearranging
  - Hover/gesture window controls

- `Graphics`
  - Screenshot tools
  - Screen recording tools
  - OCR capture tools
  - Screenshot annotation and scroll capture tools

- `Clipboard`
  - Clipboard history
  - Quick paste
  - Clipboard organization and editing

- `Player`
  - Media players
  - IPTV players
  - Audio or video playback tools

- `Other`
  - Privacy or security tools with no dedicated category
  - Specialty companion tools for specific hardware or niche workflows
  - Anything that does not fit an existing section more naturally

## Tie-breakers

When multiple categories look plausible:

1. Choose the app's primary use case, not its longest feature list.
2. Prefer the section with the closest existing peers.
3. Prefer the smaller edit that preserves current repository structure.
