# Security Policy

## What to report

Please report any app entry that:

- Has a broken/dead repository link
- Has known security risks (malware, suspicious behavior, major unresolved vulnerabilities, etc.)

## How to report

Open a GitHub issue and include:

- App name and list URL
- Evidence (official advisory, trusted report, reproducible details, screenshots, etc.)
- Why the entry should be removed or corrected

## Handling process

- Maintainer reviews and verifies the report.
- If confirmed, the app entry will be removed promptly.
- If not confirmed, maintainers may request more evidence before taking action.
